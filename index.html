Hello

I'm Brad Rivenburgh, a full-stack web developer based in the Philadelphia, PA area. I use HTML, CSS, and JavaScript to create clean and functional web applications. As someone who also has a background in instructional design, I am passionate about working with stakeholders throughout an organization at each stage of a project in order ship quality, highly-targeted applications that address client needs.

Who I am

The most important thing to know about me is that I love to solve problems. From small problems—like helping my in-laws troubleshoot their network printer (for the nth time)—to large problems—like designing, implementing, and evaluating the development process for a first-edition textbook—inventing solutions that make someone else happy just does it for me.

As you may have guessed from that last sentence, I haven’t always been a web developer. Once upon a time I:

  • Majored in History at Franklin and Marshall College
  • Earned a Master’s degree in Educational Technology at Purdue University
  • Spent years developing award winning educational products for higher-education publishers like Pearson, Cengage Learning, and Macmillan Education

Then, too, I had ample opportunity to create solutions that helped my clients meet their performance goals.

But web development, and software development in general, is special. The technologies that power the web go together like an elegant puzzle, and, for someone like me, this is something to celebrate. Whether its working as a part of a team to refactor JavaScript code in order to improve the simplicity and performance of an application, or learning an emerging framework that promises to improve the speed and quality of the development process—I’m for it. And the great need for web applications across industries means there are plenty of problems to solve.

Let's get to work.

Portfolio

Below are some of the best examples of my work to date. They are a mix of personal and professional projects.

Personal Projects

rei-search-v2

Brief description: A refactoring of my rei-search app into a full-stack React.js, Node.js, Express, and PostgreSQL app. Investors may retrieve statistics from the Census Bureau at three different geographic levels, view for-sale properties and geographic boundaries on an interactive map, and save properties for later viewing.

Audience: Active or potential real-estate investors looking to buy properties in the Philadelphia area.

My approach: I used Javascript, HTML, CSS, React, React Router, Jest, Node.js, Express, Knex.js, PostgreSQL, Mocha, Chai, Sinon, Vercel, Heroku, NPM, Git, and GitHub to build this app. It features a REST API with two endpoints (/search and /favorites) that are able to create, read, and delete properties from a database. AJAX requests were once again made to the Census and Mapbox APIs via the Fetch API and citySDK.

Lessons learned: I learned more about developing mobile-first applications with complex designs (the head-up display was particularly complex), the basics of web scraping to create data sets, the intricacies of React life-cycle methods, adapting outside libraries into React projects in a seamless way, using geographic functions in PostgreSQL queries, and unit and integration testing.

Find it on GitHub.

Link the live app.

rei-search

Brief description: An app for real-estate investors who wish to do high-level research on geographies (specifically MSAs) in which they want to invest.

Audience: Active or potential real-estate investors.

My approach: I used HTML, CSS, Javascript, JQuery, LeafletJS (map library), and CitySDK (Census FIPS codes and geoJSON shapes) to build this app. AJAX requests were made to various APIs via the Fetch API and citySDK.

Lessons learned: I furthered my knowledge of DOM abstraction, traversal, and manipulation, learned about HTTP request-response cycle, how to efficiently make AJAX requests with the Fetch API and Promises, the basics of creating and customizing interactive maps using LeafletJS, gained experience in utilizing dense and convoluted API documentation (I'm thinking of you, Census Bureau!), and added GitHub Teams to my list of development technologies.

Find it on GitHub.

Link the live app.
