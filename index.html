And the great need for web applications across industries means there are plenty of problems to solve.

Portfolio

Below are some of the best examples of my work to date.

rei-search-v2

Brief description: A refactoring of my rei-search app into a full-stack React.js, Node.js, Express, and PostgreSQL app. Investors may retrieve statistics from the Census Bureau at three different geographic levels, view for-sale properties and geographic boundaries on an interactive map, and save properties for later viewing.

Audience: Active or potential real-estate investors looking to buy properties in the Philadelphia area.

My approach: I used Javascript, HTML, CSS, React, React Router, Jest, Node.js, Express, Knex.js, PostgreSQL, Mocha, Chai, Sinon, Vercel, Heroku, NPM, Git, and GitHub to build this app. It features a REST API with two endpoints (/search and /favorites) that are able to create, read, and delete properties from a database. AJAX requests were once again made to the Census and Mapbox APIs via the Fetch API and citySDK.

Lessons learned: I learned more about developing mobile-first applications with complex designs (the head-up display was particularly complex), the basics of web scraping to create data sets, the intricacies of React life-cycle methods, adapting outside libraries into React projects in a seamless way, using geographic functions in PostgreSQL queries, and unit and integration testing.

rei-search

Brief description: An app for real-estate investors who wish to do high-level research on geographies (specifically MSAs) in which they want to invest.

Audience: Active or potential real-estate investors.

My approach: I used HTML, CSS, Javascript, JQuery, LeafletJS (map library), and CitySDK (Census FIPS codes and geoJSON shapes) to build this app. AJAX requests were made to various APIs via the Fetch API and citySDK.

Lessons learned: I furthered my knowledge of DOM abstraction, traversal, and manipulation, learned about HTTP request-response cycle, how to efficiently make AJAX requests with the Fetch API and Promises, the basics of creating and customizing interactive maps using LeafletJS, gained experience in utilizing dense and convoluted API documentation (I'm thinking of you, Census Bureau!), and added GitHub Teams to my list of development technologies.

LeetRun

Brief description: An application that allows users to record their runs and track their progress as they seek to increase their endurance, become faster, and meet their running goals.

Audience: Runners who want to use a web interface to log their runs, chart their progress, and easily filter through their past entries.

My approach: I used JavaScript, HTML, CSS, React, React Router, Chartjs, Jest, Node.js, Express, Knex.js, PostgreSQL, JWT, Bcrypt, Mocha, Chai, Sinon, Vercel, Heroku, NPM, Git, and GitHub to build this app. The REST API has four endpoints, three that enable user accounts (/login, /refresh, /users,) and one (/runs) that handles getting, inserting, updating, and deleting run entries for users.

Lessons learned: The most complicated aspect of this project was learning how to use JSON Web Tokens (JWT) for user authentication. It gave me an appreciation for how complicated user authentication is and confidence that I can handle setting up at least hobby-level auth. This project also presented a good opportunity to further my use of code layering and modularization, especially in setting up service layers to handle user-related operations.
